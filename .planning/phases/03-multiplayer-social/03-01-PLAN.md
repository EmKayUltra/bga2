    **Create `apps/client/src/lib/auth.ts`** (server-only Better Auth instance):
    - Import `betterAuth` from `better-auth`, `Pool` from `pg`, `username` from `better-auth/plugins`, `jwt` from `better-auth/plugins`
    - Configure with pg Pool using connection string: `postgres://bga2:secret@db:5432/bga2` (uses Docker service name `db`)
    - Enable `emailAndPassword: { enabled: true }`
    - Add plugins: `username()` (adds username field + sign-in-by-username), `jwt()` (adds /api/auth/token + /api/auth/jwks endpoints)
    - Set `basePath: "/api/auth"` and `baseURL: "http://localhost:5173"` (SvelteKit serves auth routes)

    **Create `apps/client/src/lib/auth-client.ts`** (browser-side auth client):
    - Import `createAuthClient` from `better-auth/svelte`
    - Import `usernameClient` from `better-auth/client/plugins`
    - Export `authClient` configured with `plugins: [usernameClient()]`
    - This provides: `authClient.signUp.email()`, `authClient.signIn.username()`, `authClient.signOut()`, `authClient.useSession()`

    **Create `apps/client/src/hooks.server.ts`** (SvelteKit server hook):
    - Import `auth` from `$lib/auth`
    - Import `svelteKitHandler` from `better-auth/svelte-kit`
    - In `handle` function: call `auth.api.getSession({ headers: event.request.headers })`, populate `event.locals.user` and `event.locals.session` if session exists
    - Return `svelteKitHandler({ event, resolve, auth })` — this intercepts `/api/auth/*` routes for Better Auth
    - Note: `building` import from `$app/environment` may be needed to guard against build-time execution

    **Update `apps/client/src/app.d.ts`** to add `locals.user` and `locals.session` types:
    ```typescript
    declare global {
      namespace App {
        interface Locals {
          user?: { id: string; email: string; name: string; username: string };
          session?: { id: string; userId: string; expiresAt: Date };
        }
      }
    }
    ```

    **Run Better Auth migration** to create auth tables (user, session, account, verification):
    ```bash
    docker compose -f apps/infra/docker-compose.yml exec client npx @better-auth/cli@latest migrate
    ```
    IMPORTANT: Better Auth manages its OWN tables (user, session, account, verification). The C# EF Core `EnsureCreated()` must NOT create conflicting tables. Better Auth tables live in the `public` schema managed exclusively by `@better-auth/cli migrate`.

    **Create `/auth/register` page** (`apps/client/src/routes/auth/register/+page.svelte`):
    - Form with fields: email, username, password, display name
    - On submit: call `authClient.signUp.email({ email, password, name, username })`
    - On success: redirect to `/` (landing page) or the `?next=` query param URL if present
    - Error display for validation failures (email taken, username taken)
    - Use Svelte 5 runes ($state for form fields, $derived for validation)

    **Create `/auth/login` page** (`apps/client/src/routes/auth/login/+page.svelte`):
    - Form with fields: username, password
    - On submit: call `authClient.signIn.username({ username, password })`
    - On success: redirect to `/` or the `?next=` query param URL
    - Error display for invalid credentials
    - Link to register page

    - Import `authClient` from `$lib/auth-client`
    - Use `authClient.useSession()` to get reactive session state
    - Show username + logout button when logged in, login/register links when not
    - Keep existing content (slot)

    **Update `Program.cs`:**
    - Add JWT Bearer authentication:
      ```csharp
      builder.Services.AddAuthentication(JwtBearerDefaults.AuthenticationScheme)
          .AddJwtBearer(options =>
          {
              // Better Auth JWKS endpoint via Docker service name
              options.Authority = "http://client:5173/api/auth";
              options.RequireHttpsMetadata = false; // Docker dev only
              options.TokenValidationParameters = new TokenValidationParameters
              {
                  ValidateIssuer = false,   // Better Auth JWT may not set standard issuer
                  ValidateAudience = false, // Same — validate signature only in dev
                  ValidateLifetime = true,
              };
          });
      builder.Services.AddAuthorization();
      ```
    - Add `app.UseAuthentication()` and `app.UseAuthorization()` BEFORE endpoint mapping
