  <behavior>
    ## getValidMoves(ctx) — factory-offer phase

    - Given factories with tiles and center with tiles, returns one ValidMove per valid (source, color, targetPatternLineRow) combination
    - A move is valid when: source zone has tiles of that color, target pattern line is not full, target pattern line is either empty or already has that color, and the wall row does not already have that color placed
    - Also includes floor-line-only moves: player can always choose to dump tiles directly to floor line (target = "player-{i}-floor-line")
    - Returns empty array when game phase is not 'factory-offer' (wall-tiling is automatic)
    - Returns empty array when game is finished

    Cases:
    - Empty factories and center → []
    - Factory with [blue, blue, red, yellow] → moves for blue (2 tiles), red (1 tile), yellow (1 tile) to each valid pattern line
    - Pattern line row 3 has 2 blue tiles (capacity 3) → blue from any source to row 3 is valid
    - Pattern line row 3 has 3 blue tiles (full) → no moves targeting row 3
    - Wall row 2 already has blue → no moves placing blue on pattern line 3 (which maps to wall row 2)
    - Center has first-player token + tiles → picking from center is valid (token auto-taken)

    ## onMove(ctx, move) — tile drafting

    - action='pick-tiles': pick all tiles of move.data.color from move.source, place on move.data.patternLineRow
    - Picking from factory: same-color tiles go to pattern line, remaining tiles of other colors go to center
    - Picking from center: same-color tiles go to pattern line, other tiles stay in center
    - First-player token: if picking from center and first-player token is present, current player takes it (goes to floor line as first item, sets hasFirstPlayerToken=true)
    - Floor line overflow: tiles that don't fit on the pattern line go to floor line; tiles that overflow floor line (max 7) go to lid
    - Turn advance: currentPlayerIndex = (current + 1) % players.length
    - Round end detection: if all factories AND center are empty after the move, set phase = 'wall-tiling'
    - Mutates ctx.state in place — NEVER reassigns ctx.state

    Cases:
    - Pick 2 blue from factory-0, place on pattern line 3 (capacity 3, has 0) → 2 blue on line, factory cleared, other tiles to center
    - Pick 3 red from factory-1, place on pattern line 2 (capacity 2, has 0) → 2 red on line, 1 red to floor
    - Pick from center with first-player token → token to floor line, hasFirstPlayerToken = true
    - Pick causes floor overflow beyond 7 → excess to lid
    - Last factory pick empties all sources → phase becomes 'wall-tiling'

    ## onRoundEnd(ctx) — wall tiling + scoring + factory refill

    Step 1: Wall tiling — for each player, for each pattern line (1-5):
    - If pattern line is full (count === row number): move the tile color to the correct wall position (standard fixed layout)
    - Score the placed tile: count contiguous horizontal run + contiguous vertical run (including the placed tile). Isolated tile = 1 point.
    - Clear the pattern line, move excess tiles to lid

    Step 2: Floor penalties — for each player:
    - Apply penalties by position: [-1, -1, -2, -2, -2, -3, -3] for positions 0-6
    - Score cannot go below 0
    - Clear the floor line, move tiles to lid
    - Clear hasFirstPlayerToken

    Step 3: End-game check:
    - If any player has a completed horizontal wall row (all 5 cells in a row are true), set finished = true
    - If finished: apply end-game bonuses for ALL players:
      - Complete row: +2 per complete row
      - Complete column: +7 per complete column
      - All 5 of one color: +10 per completed color set
    - Determine winner: highest score. Tiebreak: most complete horizontal rows.
    - Set winnerId, phase = 'finished'

    Step 4: If NOT finished, prepare next round:
    - Refill factories from bag (4 tiles each). If bag is empty, shuffle lid into bag.
    - Place first-player token in center
    - Set phase = 'factory-offer'
    - Set currentPlayerIndex to the player who has hasFirstPlayerToken (from previous round)
    - Increment round number

    Cases:
    - Pattern line 3 full with red → red placed at wall[2][wallCol], scored with adjacency
    - Tile placed next to 2 horizontal neighbors and 1 vertical → score = 3 + 2 = 5
    - Isolated tile placement → score = 1
    - Floor line with 4 tiles → penalty = -1 + -1 + -2 + -2 = -6
    - Player has score 3, floor penalty 6 → score becomes 0 (not negative)
    - Player completes wall row → game ends, bonuses applied
    - Bag empty during refill → lid shuffled into bag
    - End-game: player with 2 complete rows gets +4, 1 complete column gets +7
    - Tiebreak: both players 45 points, player A has 2 complete rows, player B has 1 → player A wins
  </behavior>
