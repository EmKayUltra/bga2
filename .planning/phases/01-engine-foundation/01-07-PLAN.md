  <action>
    1. Update `libs/engine/core/src/index.ts` barrel to export all public APIs:
       - From fsm: createGameFSM, GameFSMConfig, FSMContext, FSMEvent
       - From loader: GameLoader, RuntimeGameModel
       - From renderer: PixiAdapter, StubRenderer
       - From zones: Zone, GridZone, StackZone, etc.
       - From pieces: Piece, PieceFactory

    2. Create `apps/client/src/lib/api/gameApi.ts`:
       - `const API_BASE = 'http://localhost:8080'`
       - `async function createGame(gameId: string): Promise<{ id: string }>` — POST /games
       - `async function getGameState(sessionId: string): Promise<{ state: GameState, validMoves: ValidMove[] }>` — GET /games/:id/state
       - `async function submitMove(sessionId: string, move: Move): Promise<MoveResult>` — POST /games/:id/move
       - Error handling: network errors, 4xx responses parsed as MoveResult with errors

    3. Create `apps/client/src/lib/engine/SceneManager.ts`:
       - Orchestration layer between renderer, FSM, game model, and server API
       - Constructor takes: container (HTMLElement), gameConfig (GameConfig)
       - `async init()`:
         a. Dynamic import PixiAdapter (SSR-safe): `const { PixiAdapter } = await import('@bga2/engine-core/renderer/PixiAdapter')`
         b. Create renderer and call `await renderer.init(container, { worldWidth: 2000, worldHeight: 1500 })`
         c. Enable viewport with drag/pinch/wheel
         d. Load game model: `GameLoader.load(gameConfig)`
         e. Create FSM: `createGameFSM(config)`
         f. Create AzulScene and render initial state
       - `handlePieceClick(pieceId: string)`:
         a. If no piece selected: select piece (apply glow), fetch validMoves from current state, highlight valid destinations
         b. If piece already selected and clicking a valid destination: submit move to server, animate piece, update FSM
         c. If clicking same piece: deselect (remove glow, remove destination highlights)
       - `handleMoveResult(result: MoveResult)`:
         a. If valid: animate piece to new position (snappy overshoot), update game model, advance FSM, show scoring effects if applicable
         b. If invalid: show error feedback, deselect piece
       - `updateScene()`: re-render all pieces in their current zones based on game model state
       - `destroy()`: clean up renderer, FSM, event listeners

    4. Create `apps/client/src/lib/engine/AzulScene.ts`:
       - Azul-specific rendering logic that uses IRenderer (never PixiJS directly)
       - `constructor(renderer: IRenderer, model: RuntimeGameModel)`
       - `renderBoard()`:
         a. Create containers for: factory area, player boards
         b. Render factory displays as circles with tiles inside (styled placeholder art)
         c. Render center area
         d. Render per-player boards:
            - Pattern lines (5 rows, increasing width 1-5)
            - Wall (5x5 grid)
            - Floor line (1x7 grid)
         e. Render tiles as colored squares with subtle gradients (per user decision: styled placeholder art, not raw squares)
         f. Adaptive layout: use container width to decide layout (desktop: boards around center, mobile: your board large at bottom)
       - `renderPiece(piece: Piece)`: Create sprite using renderer.createSprite. If no asset, use procedural fallback: colored rounded rectangle with label text overlay. Set eventMode='static', set hitArea to 44px minimum.
       - `highlightValidMoves(validMoves: ValidMove[])`: Apply glow effect to all destination zones/slots that match validMoves entries. Use a distinct color (e.g., green-tinted glow) different from the selection glow (blue-tinted).
       - `clearHighlights()`: Remove all glow effects from destination zones.
       - `animatePiece(pieceId: string, targetX: number, targetY: number)`: Use renderer.animateTo with snappy overshoot easing.
       - Visual effects per user decision:
         - Scoring particle burst: emit colored particles from scored positions (simple renderer.createSprite particles with fade-out)
         - Placement feedback: brief scale pulse when tile locks into wall position
         - Ambient board glow: subtle glow filter on board container (low strength, warm color)

    NOTE: All rendering goes through IRenderer methods. AzulScene must NOT import pixi.js directly. This is what proves the renderer abstraction works. If you can swap PixiAdapter for StubRenderer and AzulScene still runs without errors, the abstraction is correct.
  </action>
