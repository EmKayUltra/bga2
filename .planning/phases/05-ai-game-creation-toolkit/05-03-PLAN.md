Hive game rules (from research):
- 2-player abstract strategy
- 5 piece types: Queen Bee (1), Beetle (2), Grasshopper (3), Spider (2), Soldier Ant (3) — per player
- No pre-defined board — pieces placed adjacent to existing pieces, first piece at (0,0)
- Queen Bee MUST be placed by turn 4 (forced placement on turn 4 if not yet placed)
- Win: opponent's Queen Bee is completely surrounded (all 6 hex neighbors occupied by any color)
- One Hive Rule: removing a piece for movement must not disconnect the hive
- Freedom-to-Move: a piece can only slide to a neighbor if there's no "gate" (two pieces blocking the gap)
- Beetle: can climb on top of adjacent pieces (stacking); top-of-stack controls the hex
- Grasshopper: jumps in a straight line over one or more occupied hexes, landing on the first empty hex
- Spider: moves exactly 3 spaces along the hive perimeter (sliding constraint applies each step)
- Soldier Ant: moves to any reachable empty hex along the hive perimeter (unlimited distance)
- Queen Bee: moves exactly 1 space (standard sliding move)

```typescript
/**
 * Axial hex coordinate utilities for Hive.
 * Source: https://www.redblobgames.com/grids/hexagons/
 *
 * Uses axial coordinates (q, r) where the third coordinate s = -q - r.
 * Pointy-top orientation.
 */

/** All 6 neighbor direction offsets (pointy-top hexes). */
export const HEX_DIRECTIONS: Array<{ q: number; r: number }> = [
  { q: 1, r: 0 },
  { q: 1, r: -1 },
  { q: 0, r: -1 },
  { q: -1, r: 0 },
  { q: -1, r: 1 },
  { q: 0, r: 1 },
];

/** Create a string key from hex coordinates for use as Map/object keys. */
export function hexKey(q: number, r: number): string {
  return q + ',' + r;
}

/** Parse a hex key back to coordinates. */
export function parseHexKey(key: string): { q: number; r: number } {
  const parts = key.split(',');
  return { q: parseInt(parts[0], 10), r: parseInt(parts[1], 10) };
}

/** Get all 6 neighbor coordinates of a hex. */
export function hexNeighbors(q: number, r: number): Array<{ q: number; r: number }> {
  return HEX_DIRECTIONS.map(function (d) {
    return { q: q + d.q, r: r + d.r };
  });
}

/** Manhattan distance between two axial hex coordinates. */
export function hexDistance(a: { q: number; r: number }, b: { q: number; r: number }): number {
  const dq = Math.abs(a.q - b.q);
  const dr = Math.abs(a.r - b.r);
  const ds = Math.abs((-a.q - a.r) - (-b.q - b.r));
  return Math.max(dq, dr, ds);
}

/**
 * Convert axial hex coordinate to pixel position (pointy-top orientation).
 * @param size Half the hex width (distance from center to corner).
 */
export function hexToPixel(q: number, r: number, size: number): { x: number; y: number } {
  return {
    x: size * (Math.sqrt(3) * q + Math.sqrt(3) / 2 * r),
    y: size * (3 / 2 * r),
  };
}

/**
 * Check if all pieces in the occupied set remain connected after removing one hex.
 * Uses BFS from any remaining piece. Returns true if connected.
 *
 * @param occupiedKeys Set of hex key strings that are occupied (e.g., "0,0", "1,-1")
 * @param excludeKey The hex key to exclude (simulating removal of that piece)
 */
export function isConnectedWithout(occupiedKeys: Set<string>, excludeKey: string): boolean {
  const remaining: string[] = [];
  occupiedKeys.forEach(function (key) {
    if (key !== excludeKey) remaining.push(key);
  });

  if (remaining.length === 0) return true;
  if (remaining.length === 1) return true;

  const visited = new Set<string>();
  const queue: string[] = [remaining[0]];
  visited.add(remaining[0]);

  while (queue.length > 0) {
    const current = queue.shift() as string;
    const coord = parseHexKey(current);
    const neighbors = hexNeighbors(coord.q, coord.r);

    for (let i = 0; i < neighbors.length; i++) {
      const nk = hexKey(neighbors[i].q, neighbors[i].r);
      if (!visited.has(nk) && occupiedKeys.has(nk) && nk !== excludeKey) {
        visited.add(nk);
        queue.push(nk);
      }
    }
  }

  return visited.size === remaining.length;
}

/**
 * Check if a piece can physically slide from one hex to an adjacent hex.
 * Freedom-to-Move rule: there must not be two pieces on both sides of the gap.
 *
 * For a piece at (fromQ, fromR) sliding to neighbor (toQ, toR):
 * The two hexes adjacent to BOTH from and to must not BOTH be occupied.
 *
 * @param occupiedKeys Set of occupied hex keys (NOT including the moving piece's current position)
 * @param fromQ Source Q
 * @param fromR Source R
 * @param toQ Target Q
 * @param toR Target R
 */
export function canSlide(
  occupiedKeys: Set<string>,
  fromQ: number,
  fromR: number,
  toQ: number,
  toR: number
): boolean {
  // The direction from 'from' to 'to'
  const dq = toQ - fromQ;
  const dr = toR - fromR;

  // Find the two common neighbors of from and to
  // These are the hexes that share edges with both from and to
  const fromNeighbors = hexNeighbors(fromQ, fromR);
  const toNeighbors = hexNeighbors(toQ, toR);

  const commonNeighbors: Array<{ q: number; r: number }> = [];
  for (let i = 0; i < fromNeighbors.length; i++) {
    for (let j = 0; j < toNeighbors.length; j++) {
      if (fromNeighbors[i].q === toNeighbors[j].q && fromNeighbors[i].r === toNeighbors[j].r) {
        // Exclude from and to themselves
        if (!(fromNeighbors[i].q === fromQ && fromNeighbors[i].r === fromR) &&
            !(fromNeighbors[i].q === toQ && fromNeighbors[i].r === toR)) {
          commonNeighbors.push(fromNeighbors[i]);
        }
      }
    }
  }

  // If both common neighbors are occupied, the piece cannot slide through
  if (commonNeighbors.length === 2) {
    const key1 = hexKey(commonNeighbors[0].q, commonNeighbors[0].r);
    const key2 = hexKey(commonNeighbors[1].q, commonNeighbors[1].r);
    if (occupiedKeys.has(key1) && occupiedKeys.has(key2)) {
      return false;
    }
  }

  return true;
}
```

The actual implementation must be detailed and correct. Implement all 5 piece movement algorithms:

**Queen Bee movement:** Find all adjacent empty hexes where canSlide is true and the piece remains adjacent to at least one other piece after moving (must touch the hive).

**Beetle movement:** Find all 6 adjacent hexes. Can move to empty hexes (sliding constraint applies at ground level) or climb onto occupied hexes (no sliding constraint when climbing up/down).

**Grasshopper movement:** For each of the 6 hex directions, check if the adjacent hex in that direction is occupied. If yes, continue in that direction until an empty hex is found. That empty hex is a valid target.

**Spider movement:** BFS to depth exactly 3, following the hive perimeter. Each step must be a slide (canSlide check), the piece must stay adjacent to the hive at each step, and no hex can be visited twice in the same path.

**Soldier Ant movement:** BFS of unlimited depth along the hive perimeter. Each step must be a slide (canSlide check), the piece must stay adjacent to the hive. Can reach any empty hex reachable by consecutive slides.

**Test categories:**

**A. Placement rules:**
- First move: only valid move is place at (0,0)
- Second move (other player): must place adjacent to first piece
- After first moves: can only place adjacent to own pieces (not touching enemy)
- Queen Bee must be placed by turn 4: if turn 4 and queen not placed, only queen placement moves returned
- Cannot place when hand is empty for that piece type

**B. Movement rules (require queen placed):**
- Cannot move any piece until own queen is placed
- Queen Bee moves exactly 1 space with sliding constraint
- Beetle moves 1 space, can climb onto occupied hex (stack level increases)
- Beetle on top of stack: can move to any adjacent (even occupied) hex
- Grasshopper: must jump over at least 1 piece in a straight line
- Grasshopper: lands on first empty hex in the jump direction
- Spider: moves exactly 3 spaces along hive perimeter
- Soldier Ant: can reach any empty hex along hive perimeter

**C. One Hive Rule:**
- Piece that would disconnect the hive cannot be moved
- Piece at a "hinge" point has no movement options
- Piece on the edge of the hive can move (does not disconnect)

**D. Freedom-to-Move (sliding):**
- Piece cannot slide through a gap blocked by two adjacent pieces
- Piece can slide when only one side of the gap is blocked

**E. Win condition:**
- Game ends when a queen is completely surrounded (all 6 neighbors occupied)
- Winner is the player whose queen is NOT surrounded
- If both queens surrounded simultaneously, handle appropriately
