    **StubRenderer.ts:**
    - Implements every IRenderer method as a no-op or minimal tracking implementation
    - init: stores options, resolves immediately
    - createSprite: returns { id: uuid, type: 'sprite' } as ISpriteHandle
    - createContainer: returns { id: uuid, type: 'container' } as IContainerHandle
    - addToStage/removeFromStage: push/filter from internal array (useful for testing what's on stage)
    - setInteractive: no-op
    - onPointerDown: stores callback in a map (tests can trigger it)
    - applyGlow/removeGlow: no-op
    - animateTo: resolves immediately with no delay
    - enableViewport: returns { id: uuid, type: 'viewport' } as IViewportHandle
    - destroy: clears internal state
    - Add helper methods for testing: `getStageObjects()`, `triggerPointerDown(handleId)`, `getGlowState(handleId)`

    **PixiAdapter.ts:**
    - `import '@mszu/pixi-ssr-shim'` as FIRST import (SSR safety per research Pitfall 1)
    - `import { Application, Sprite, Container, Texture, Graphics } from 'pixi.js'`
    - `import { GlowFilter } from 'pixi-filters'`
    - `import { Viewport } from 'pixi-viewport'`

    - init(): Create Application with `await app.init({ resizeTo: container, preference: 'webgl', antialias: true, background: 0xfafaf8 })`. Append app.canvas to container. Light + clean theme background per user decision.

    - createSprite(textureId): If texture exists in cache, create Sprite from it. Otherwise create a colored Graphics placeholder. Set `eventMode = 'static'` for interaction. Set hitArea to minimum 44px bounds (ENG-09). Return as ISpriteHandle wrapping the PixiJS DisplayObject.

    - createContainer(): Create PixiJS Container, return as IContainerHandle.

    - addToStage/removeFromStage: Add/remove from viewport (not app.stage directly — all game objects live inside the viewport for zoom/pan).

    - setInteractive(handle, enabled): Set `eventMode` to 'static' (enabled) or 'none' (disabled) on the underlying PixiJS object.

    - onPointerDown(handle, cb): Add `pointerdown` event listener to underlying PixiJS object.

    - applyGlow(handle, color, strength): Create GlowFilter with specified color and outerStrength. Set on sprite.filters array. Per user decision: selected piece gets GlowFilter.

    - removeGlow(handle): Clear sprite.filters to empty array.

    - animateTo(handle, x, y, opts): Animate position using requestAnimationFrame loop with snappy overshoot easing per user decision. Use a custom easing function: `easeOutBack(t) = 1 + 2.70158 * pow(t-1, 3) + 1.70158 * pow(t-1, 2)`. Return Promise that resolves when animation completes.

    - enableViewport(options): Create pixi-viewport Viewport with screenWidth, screenHeight, worldWidth, worldHeight, and `events: this.app.renderer.events` (CRITICAL per research Pitfall 2). Chain `.drag().pinch().wheel().decelerate()`. Add viewport to app.stage. Return as IViewportHandle.

    - resize(width, height): Update app.renderer.resize and viewport dimensions.

    - destroy(): Call app.destroy(true) to clean up WebGL context.

    - Internal handle map: Map<string, PixiJS DisplayObject> to look up PixiJS objects from handle IDs.

    - 44px touch target enforcement (ENG-09): When creating any interactive sprite, set `hitArea` to at least a 44x44 Rectangle. If the sprite's natural bounds are smaller, expand the hitArea.

    **index.ts:** barrel export PixiAdapter, StubRenderer.

    NOTE: PixiAdapter tests will be minimal (type-check that it implements IRenderer) since PixiJS requires a browser/WebGL context. Full PixiAdapter verification happens in Plan 07 (integration). StubRenderer gets thorough unit tests.
