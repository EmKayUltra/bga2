    **Create `apps/server/Services/FriendService.cs`:**
    Depends on `GameDbContext`, `ILogger<FriendService>`.

    - `SearchUsers(string query, string excludeUserId) → List<UserSearchResult>`
      - Query Better Auth's `user` table via raw SQL: `SELECT id, username, name FROM "user" WHERE username ILIKE @q AND id != @exclude LIMIT 10`
      - Returns: userId, username, displayName
      - Does NOT return users who have blocked the searcher

    - `SendRequest(string requesterId, string addresseeUsername) → FriendRequestResult`
      - Look up addressee by username from `user` table
      - Check no existing friendship (pending or accepted) between the pair (check both directions)
      - Check addressee has not blocked requester
      - Create Friendship row with Status = Pending
      - Return success/error

    - `AcceptRequest(string addresseeId, Guid friendshipId) → bool`
      - Verify the friendship exists, status is Pending, and addressee matches
      - Update status to Accepted, set AcceptedAt = now
      - Return success

    - `DeclineRequest(string addresseeId, Guid friendshipId) → bool`
      - Verify ownership, delete the Friendship row

    - `RemoveFriend(string userId, Guid friendshipId) → bool`
      - Either party can remove; delete the Friendship row

    - `BlockUser(string userId, string blockedUserId) → bool`
      - If friendship exists, update status to Blocked
      - If no friendship, create one with status Blocked

    - `GetFriends(string userId) → List<FriendInfo>`
      - Query accepted friendships where user is requester OR addressee
      - Look up friend's username/displayName from `user` table
      - Include online status (see below)
      - Return: friendshipId, userId, username, displayName, avatar, isOnline, friendSince

    - `GetPendingRequests(string userId) → (List<FriendRequestInfo> incoming, List<FriendRequestInfo> outgoing)`
      - Incoming: addresseeId == userId, status == Pending
      - Outgoing: requesterId == userId, status == Pending

    **Online/offline presence** (polling-based per CONTEXT.md decision):
    - Add a simple `LastSeen` tracking approach:
      - Store last-seen timestamps in a static `ConcurrentDictionary<string, DateTime>` in FriendService (in-memory, lost on restart — acceptable for dev)
      - Update on any authenticated API call (middleware or endpoint filter)
      - `IsOnline(string userId)` → true if lastSeen within 60 seconds
    - Alternative: add a `LastSeenAt` column to UserProfile table (Plan 03). Either approach works; in-memory is simpler for Phase 3.
    - For production: would use Redis or DynamoDB TTL. In-memory ConcurrentDictionary is fine for dev.

    **Create `apps/server/Services/InviteService.cs`:**
    Depends on `IConfiguration`, `GameDbContext`.

    - `CreateInviteToken(Guid tableId, string hostUserId) → string`
      - Generate token: HMACSHA256 over `{tableId}:{expiresAt}` using a server secret (config key `InviteSecret`, default "bga2-dev-invite-secret")
      - Token format: `{tableId}:{expiresAt}:{signature}` base64url encoded
      - Expiry: 24 hours from creation

    - `ValidateInviteToken(string token) → InviteValidation`
      - Decode base64url, split parts, verify HMAC signature
      - Check not expired
      - Return: { valid: true, tableId } or { valid: false, error: "..." }

    **Create `apps/server/Endpoints/FriendEndpoints.cs`:**
    - `app.MapGroup("/friends").WithTags("Friends")`
    - `GET /friends` → GetFriends (requires auth)
    - `GET /friends/requests` → GetPendingRequests (requires auth)
    - `GET /friends/search?q={query}` → SearchUsers (requires auth)
    - `POST /friends/request` → SendRequest (requires auth, body: { username })
    - `POST /friends/{id}/accept` → AcceptRequest (requires auth)
    - `POST /friends/{id}/decline` → DeclineRequest (requires auth)
    - `DELETE /friends/{id}` → RemoveFriend (requires auth)
    - `POST /friends/block` → BlockUser (requires auth, body: { userId })

    **Create `apps/server/Endpoints/InviteEndpoints.cs`:**
    - `app.MapGroup("/invites").WithTags("Invites")`
    - `POST /invites` → CreateInviteToken (requires auth, body: { tableId })
    - `GET /invites/{token}/validate` → ValidateInviteToken (no auth — link recipients may not be logged in yet)

    **Update `apps/server/Program.cs`:**
    - Register FriendService and InviteService as scoped
    - Add `app.MapFriendEndpoints()` and `app.MapInviteEndpoints()`
    - Add last-seen middleware: after authentication, update FriendService.UpdateLastSeen(userId)
