<objective>
Build the user-facing UI for the async notification system: notification preferences on the settings page, async filter and My Games section in the lobby, timer countdown display on the game page with green/yellow/red color shifts, mutual pause request/accept flow, in-app notification badge, and push permission auto-prompt on first async game.

Purpose: Without the UI layer, all the backend notification infrastructure is invisible to players. This plan makes async mode tangible — players see timers, configure how they want to be notified, manage their active games, and pause when life gets in the way.

Output: Extended settings page, extended lobby page, extended game page with timer + pause, push permission auto-prompt, visual verification.
</objective>

**2. Lobby page — My Games section:**
At the TOP of the lobby page (above the public table list), add a "My Games" section that is visible only when the user is authenticated and has active async games. Call `getMyGames()` on mount (with the existing polling interval or a separate 30s poll). Display as a compact list/card layout:
- Each game shows: display name, opponent names, timer mode badge, time remaining (formatted as "2h 15m" or "3d 12h"), "Your Turn" badge (green) or "Waiting" badge (gray), link to /game/{sessionId}
- If isPaused, show "Paused" badge instead of timer
- Sort: "Your Turn" games first, then by deadline (most urgent first)

**3. Game page — timer countdown display (per locked decision):**
In the existing toolbar area (near turn status display), add a timer countdown for async games:
- On mount, check if the game table is async. Add a new API endpoint or extend the existing game state response to include turnDeadline and isAsync info. The simplest approach: extend GET /games/{id}/state to also return async metadata (turnDeadline, isAsync, isPaused) by looking up the GameTable for this session. Alternatively, include turnDeadline in the game state JSON itself.
- **Recommended approach:** Add an endpoint GET /tables/by-session/{sessionId} that returns table metadata (isAsync, timerMode, turnDeadline, isPaused, pauseRequestedByUserId). The game page calls this once on mount and caches. Timer updates are client-side via setInterval.
- Display: compact countdown text next to turn status, e.g., "23:45:12" or "2d 11h". Color shifts:
  - Green (#22c55e): > 50% of total time remaining
  - Yellow (#eab308): 25-50% remaining
  - Red (#ef4444): < 25% remaining (pulsing animation via CSS)
- Timer updates every second via setInterval. Clean up in onDestroy.

**4. Game page — pause request flow (per locked decision):**
Add a "Request Pause" button visible to any player in an async game (only when game is NOT already paused and no pause request is pending). Flow:
- Player A clicks "Request Pause" -> POST /tables/{tableId}/pause-request (new endpoint)
- Server sets PauseRequestedByUserId = requesting userId on GameTable. Notify other player(s) via AppSync.
- Other player sees "Pause Requested by {name}" banner with "Accept Pause" / "Decline" buttons
- Accept: POST /tables/{tableId}/pause-accept -> server sets IsPaused = true, TurnDeadline = null
- Decline: POST /tables/{tableId}/pause-decline -> server clears PauseRequestedByUserId
- When paused: show "Game Paused" banner. Either player can "Resume" -> POST /tables/{tableId}/resume -> server sets IsPaused = false, recalculates TurnDeadline from TimerMode

Add these 4 endpoints to LobbyEndpoints (pause-request, pause-accept, pause-decline, resume) with LobbyService methods. All require auth. Validate that the requesting user is a player in the table.

**5. In-app badge in nav bar:**
In +layout.svelte, add a notification badge on the lobby link (or a dedicated nav icon) showing the count of async games where it's the player's turn. Use `getMyGames()` and count items where `isMyTurn === true`. Poll every 60s when authenticated. Display as a small red circle with the count number (e.g., the classic notification badge). If count is 0, hide the badge entirely.

**6. Push permission auto-prompt on first async game:**
In the game/[id]/+page.svelte, after mount: if the game is async AND push permission is 'default' (not yet asked) AND the user is authenticated:
- Call `subscribeToPush(VITE_VAPID_PUBLIC_KEY)` which triggers the browser permission dialog

- On iOS: check `isInstalledPWA()` first. If NOT installed, show a soft message: "Install BGA2 to your home screen to receive push notifications" instead of prompting.
