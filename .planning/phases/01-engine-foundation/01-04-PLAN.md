Build the game loader that parses game.json into a runtime model, and the zone/piece primitive classes that represent game components at runtime.

Purpose: The engine needs to understand games. The loader transforms the declarative game.json into live objects (zones, pieces) that the FSM can operate on and the renderer can display. This is the bridge between data and runtime.
Output: GameLoader, Zone hierarchy (5 types), Piece class, and factories for both.
</objective>

  <behavior>
    - Test 1: GridZone stores pieces by (row, col) coordinates; getPieceAt(row, col) returns correct piece
    - Test 2: GridZone rejects placement at occupied cell
    - Test 3: StackZone pushes and pops pieces in LIFO order
    - Test 4: StackZone respects capacity limit (push on full stack fails)
    - Test 5: HandZone holds pieces without positional ordering (set semantics)
    - Test 6: DeckZone supports shuffle (changes piece order) and draw (removes top piece)
    - Test 7: DiscardZone accepts pieces and allows retrieval of all pieces
    - Test 8: ZoneFactory creates correct zone type from ZoneDef
    - Test 9: All zone types report their piece count correctly
  </behavior>
  <action>
    Create zone class hierarchy:

    **Zone.ts** — Base Zone class:
    - `id: string`, `type: ZoneType`, `owner: 'player' | 'shared'`, `capacity?: number`
    - Abstract methods: `addPiece(piece: Piece, position?)`, `removePiece(pieceId: string): Piece | null`, `getPieces(): Piece[]`, `getPieceCount(): number`, `isFull(): boolean`
    - Concrete subclasses inline in same file (they're small):

    **GridZone** extends Zone:
    - Internal grid: `Map<string, Piece>` keyed by "row,col"
    - `addPiece(piece, { row, col })` — place at grid position, reject if occupied
    - `getPieceAt(row, col)` — return piece at position
    - `removePiece(pieceId)` — find and remove from grid
    - rows, cols from ZoneDef

    **StackZone** extends Zone:
    - Internal array (top = last element)
    - `push(piece)` — add to top, reject if capacity exceeded
    - `pop()` — remove from top
    - `peek()` — view top without removing

    **HandZone** extends Zone:
    - Internal Set of pieces (no positional order)
    - `addPiece(piece)` — add to hand
    - `removePiece(pieceId)` — remove by id
    - `hasPiece(pieceId)` — check membership

    **DeckZone** extends Zone:
    - Internal array
    - `shuffle()` — Fisher-Yates shuffle in place
    - `draw()` — remove and return top piece
    - `drawN(n)` — draw multiple
    - `addToBottom(piece)` — add to bottom of deck

    **DiscardZone** extends Zone:
    - Internal array
    - `addPiece(piece)` — add to discard
    - `getAllPieces()` — return all pieces
    - `clear()` — empty the discard pile (for reshuffling into deck)

    **ZoneFactory.ts:**
    - `createZone(def: ZoneDef): Zone` — switch on def.type to instantiate correct subclass

    **index.ts:** barrel export all zone classes and factory.
  </action>

    <automated>npx vitest run libs/engine/core/src/zones/ --reporter=verbose 2>&1 | tail -20</automated>
  </verify>
  <done>All 5 zone types instantiate from ZoneDef. GridZone addresses by (row,col). StackZone has LIFO push/pop. DeckZone shuffles and draws. All tests pass (minimum 9 tests).</done>
