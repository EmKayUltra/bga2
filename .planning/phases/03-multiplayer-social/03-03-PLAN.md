    **Create `apps/server/Data/UserProfile.cs`:**
    C# needs a profile extension table since Better Auth owns the user table. This stores avatar choice and privacy setting.
    ```csharp
    public class UserProfile
    {
        public string UserId { get; set; } = "";       // PK, references Better Auth user.id
        public string Avatar { get; set; } = "default"; // Preset avatar identifier
        public bool IsPublic { get; set; } = true;      // Profile visibility
        public DateTime? UsernameChangedAt { get; set; } // Tracks last username change for cooldown
        public DateTime CreatedAt { get; set; }
        public DateTime UpdatedAt { get; set; }
    }
    ```
    Add `DbSet<UserProfile>` to GameDbContext with UserId as primary key.

    **Create `apps/server/Services/ProfileService.cs`:**
    Depends on `GameDbContext`, `ILogger<ProfileService>`.

    - `GetProfile(string username) → ProfileResponse?`
      - Query Better Auth's `user` table via raw SQL: `SELECT id, name, username, "createdAt" FROM "user" WHERE username = @username`
      - Join with UserProfile for avatar/privacy
      - If no UserProfile row exists, return defaults (avatar="default", isPublic=true)
      - If profile is private and viewer is not the user, return limited info
      - Aggregate stats from MatchResult: total games, wins, win rate
      - Return: username, displayName, avatar, memberSince, gamesPlayed, winRate, isPublic

    - `GetMatchHistory(string userId, int page, int pageSize) → List<MatchHistoryItem>`
      - Query MatchResults for userId, ordered by CompletedAt descending
      - For each result, look up opponent names via other MatchResults with same SessionId
      - Return: gameId, result (won/lost), score, rank, opponents, completedAt, playerCount

    - `UpdateProfile(string userId, UpdateProfileRequest req) → bool`
      - Upsert UserProfile: avatar, isPublic
      - Validate avatar is in the allowed preset list

    - `UpdateUsername(string userId, string newUsername) → UsernameChangeResult`
      - Validates newUsername: 3-20 chars, alphanumeric + underscores only
      - Checks uniqueness via raw SQL against Better Auth's `user` table: `SELECT COUNT(*) FROM "user" WHERE username = @newUsername AND id != @userId`
      - Checks cooldown: loads UserProfile.UsernameChangedAt — if within 30 days, return error with days remaining
      - Updates username in Better Auth's `user` table via raw SQL: `UPDATE "user" SET username = @newUsername WHERE id = @userId`
      - Updates UserProfile.UsernameChangedAt = DateTime.UtcNow
      - Returns: `UsernameChangeResult(bool Success, string? Error)`

    - `RecordMatchResults(Guid sessionId, string gameId, GameEndData endData) → void`
      - Called when a game ends (from GameService)
      - Creates a MatchResult per player with won/score/rank/playerCount
      - endData contains player scores and rankings extracted from game state

    **Preset avatar list** (defined as a static array in ProfileService):
    16 preset avatars: `["knight", "wizard", "dragon", "phoenix", "castle", "crown", "shield", "sword", "dice", "pawn", "rook", "queen", "joker", "crystal", "scroll", "compass"]`
    These are identifiers — the client maps them to emoji or SVG icons.

    **Create `apps/server/Endpoints/SocialEndpoints.cs`:**
    - `app.MapGroup("/social").WithTags("Social")`
    - `GET /social/profile/{username}` → GetProfile (public, no auth)
    - `GET /social/profile/{username}/history` → GetMatchHistory (public if profile is public)
    - `PUT /social/profile` → UpdateProfile (requires auth — updates own profile)
    - `PUT /social/profile/username` → UpdateUsername (requires auth, body: `{ username: string }`) — returns 200 on success, 409 if username taken, 429 if cooldown active (with `retryAfterDays` in response body)
    - `GET /social/avatars` → returns the preset avatar list (public)

    **Update `apps/server/Services/GameService.cs`:**
    - When a game ends (detected by `finished == true` in state after move), call `ProfileService.RecordMatchResults`
    - Extract player scores and rankings from the game state JSON
    - This hooks into the existing `ValidateAndApplyMove` flow: after applying a move, check if `finished` field is true in the new state

    **Update `apps/server/Program.cs`:**
    - Register `ProfileService` as scoped
    - Add `app.MapSocialEndpoints()`
