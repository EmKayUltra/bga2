    **Create `apps/server/Services/LobbyService.cs`:**

    Depends on `GameDbContext`, `GameService`, `ILogger<LobbyService>`. Methods:

    - `CreateTable(string userId, string displayName, CreateTableRequest req) → GameTable`
      - Creates a GameTable with host info, game settings (gameId, minPlayers, maxPlayers, isPrivate, passwordHash)
      - Auto-adds the host as first TablePlayer (seatIndex=0, isReady=true)
      - Returns the created table

    - `ListTables() → List<TableListItem>`
      - Returns tables with Status == Waiting, ordered by CreatedAt descending
      - Includes player count and host display name
      - Excludes private tables from public listing (they're join-by-invite/link only)

    - `GetTable(Guid tableId) → TableDetail?`
      - Returns table with all players (for waiting room display)
      - Returns null if not found

    - `JoinTable(Guid tableId, string userId, string displayName, string? password) → JoinResult`
      - Validates: table exists, status == Waiting, not full, not already joined
      - If private with password: verify password (BCrypt or simple hash comparison)
      - Adds TablePlayer with next available seatIndex
      - Returns success/error

    - `LeaveTable(Guid tableId, string userId) → bool`
      - Removes TablePlayer; if host leaves, delete table or transfer host

    - `StartGame(Guid tableId, string userId) → StartGameResult`
      - Validates: caller is host, enough players (>= minPlayers)
      - Calls GameService.CreateGame(gameId, playerNames from TablePlayers)
      - Updates table: Status = Playing, SessionId = new game session ID
      - Returns sessionId for redirect

    - `QuickPlay(string userId, string displayName, string gameId) → QuickPlayResult`
      - Finds first public Waiting table for gameId with open seats → join it
      - If none available → create new public table and return tableId
      - Return: either { joined: true, tableId } or { created: true, tableId }

    **Define request/response records** in LobbyEndpoints.cs or a separate Models file:
    - `CreateTableRequest(string GameId, string DisplayName, int MinPlayers, int MaxPlayers, bool IsPrivate, string? Password)`
    - `TableListItem(Guid Id, string GameId, string DisplayName, string HostName, int PlayerCount, int MaxPlayers, DateTime CreatedAt)`
    - `TableDetail(Guid Id, string GameId, string DisplayName, bool IsPrivate, TableStatus Status, Guid? SessionId, List<TablePlayerInfo> Players)`
    - `TablePlayerInfo(string UserId, string DisplayName, int SeatIndex, bool IsReady)`
    - `JoinResult(bool Success, string? Error)`
    - `StartGameResult(bool Success, Guid? SessionId, string? Error)`
    - `QuickPlayResult(bool Joined, bool Created, Guid TableId)`

    **Create `apps/server/Endpoints/LobbyEndpoints.cs`:**
    - Follow MapGroup pattern: `app.MapGroup("/tables").WithTags("Lobby")`
    - `GET /tables` → ListTables (public, no auth)
    - `GET /tables/{id}` → GetTable (no auth — needed for waiting room polling)
    - `POST /tables` → CreateTable (requires auth)
    - `POST /tables/{id}/join` → JoinTable (requires auth)
    - `POST /tables/{id}/leave` → LeaveTable (requires auth)
    - `POST /tables/{id}/start` → StartGame (requires auth, host only)
    - `POST /tables/quick-play` → QuickPlay (requires auth)
    - Extract userId from JWT claims: `context.User.FindFirst(ClaimTypes.NameIdentifier)?.Value`

    **Update `Program.cs`:**
    - Register `LobbyService` as scoped
    - Add `app.MapLobbyEndpoints()`
