    - Test 3: ZoneDef has id, type (enum: grid | stack | hand | deck | discard), plus type-specific fields (capacity, rows, cols, owner)

    - Test 7: ValidMove has action, source, target, pieceId fields (no playerId — implied)

  <action>
    Create the type definition files:

    **renderer.ts** — IRenderer interface per research Pattern 1:
    - Lifecycle: init(container: HTMLElement, options: RendererOptions): Promise<void>, destroy(): void, resize(width: number, height: number): void
    - Scene graph: createSprite(textureId: string): ISpriteHandle, createContainer(): IContainerHandle, addToStage(handle: ISceneHandle): void, removeFromStage(handle: ISceneHandle): void
    - Interaction: setInteractive(handle: ISpriteHandle, enabled: boolean): void, onPointerDown(handle: ISpriteHandle, cb: () => void): void
    - Effects: applyGlow(handle: ISpriteHandle, color: number, strength: number): void, removeGlow(handle: ISpriteHandle): void, animateTo(handle: ISpriteHandle, x: number, y: number, opts: AnimateOptions): Promise<void>
    - Viewport: enableViewport(options: ViewportOptions): IViewportHandle
    - Handle types: ISpriteHandle, IContainerHandle, IViewportHandle, ISceneHandle (union)
    - Options: RendererOptions (worldWidth, worldHeight, background), AnimateOptions (duration, easing), ViewportOptions (screenWidth, screenHeight, worldWidth, worldHeight)

    **game-config.ts** — GameConfig type per research Pattern 3:
    - GameConfig: id, version, title, players ({min, max}), zones (ZoneDef[]), pieces (PieceDef[]), turnOrder (TurnOrder), hooks (HookRef)
    - HookRef: file (string), events (string[])

    **zones.ts** — Zone types:
    - ZoneType enum: 'grid' | 'stack' | 'hand' | 'deck' | 'discard'
    - ZoneDef: id, type (ZoneType), capacity?, rows?, cols?, owner? ('player' | 'shared'), position? ({x, y}), render? (ZoneRenderConfig)
    - ZoneRenderConfig: shape?, size?, cellSize?

    **pieces.ts** — Piece types:
    - PieceType: 'tile' | 'card' | 'token' | 'die'
    - PieceDef: id, type (PieceType), asset? (string), fallback ({shape: string, color: string, label: string}), properties? (Record<string, unknown>)
    - PieceState: id, defId, zoneId, position? ({row, col} | {index}), ownerId?, state? (Record<string, unknown>)

    **hooks.ts** — Hook contract:
    - HookContext: state (GameState), currentPlayer (string), round (number), players (PlayerInfo[])
    - HookFunctions: getValidMoves(ctx: HookContext): ValidMove[], onMove(ctx: HookContext, move: Move): void, onRoundEnd(ctx: HookContext): void
    - PlayerInfo: id, name, index

    **moves.ts** — Move types:
    - Move: playerId, action (string), source? (string), target? (string), pieceId? (string), data? (Record<string, unknown>)
    - ValidMove: action (string), source? (string), target? (string), pieceId? (string), description? (string)
    - MoveResult: valid (boolean), newState? (GameState), validMoves? (ValidMove[]), errors? (string[])

    **state.ts** — Game state:
    - GameState: id (string), gameId (string), version (number), phase (string), currentPlayerIndex (number), players (PlayerState[]), zones (Record<string, ZoneState>), round (number), finished (boolean), winnerId? (string)
    - PlayerState: id, name, score (number), data (Record<string, unknown>)
    - ZoneState: id, pieces (PieceState[])

    **index.ts** — Barrel export all types from all files.

    All types are pure interfaces/type aliases — no runtime code, no classes.
  </action>
